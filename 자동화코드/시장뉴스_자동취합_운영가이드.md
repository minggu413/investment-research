# 시장뉴스 자동 취합 운영가이드

## 목표

- `시장뉴스 취합 템플릿.md` 기준으로 시장뉴스를 작성함.
- 결과는 `시장뉴스/` 폴더에 날짜별 Markdown 파일로 저장함.
- 저장 파일만 GitHub `main` 브랜치에 업로드함.
- 인증 파일, 토큰, 로컬 DB, 개인 폴더는 GitHub에 올리지 않음.
- 자동 실행은 평일 오전 5시 한국시간으로 맞춤.

## 실행 기준

- 기준 템플릿: `시장뉴스 취합 템플릿.md`
- 저장 위치: `시장뉴스/YYYY-MM-DD_시장뉴스.md`
- 기본 기사 수: 40개
- 문체: 음슴체
- 기사별 형식: 제목, URL, 요약 3줄, 투자자 해석 2줄
- 카테고리: 템플릿의 `## 카테고리` 목록을 기준으로 적용함.

## GitHub 업로드 기준

- 작업 전 `main` 브랜치에서 최신 상태인지 확인함.
- 결과 파일 생성 후 `git status`로 변경 파일 확인함.
- `시장뉴스/YYYY-MM-DD_시장뉴스.md`와 필요한 문서 변경만 커밋함.
- 인증 파일, 토큰, `.env`, 로컬 DB, 개인 폴더는 커밋하지 않음.
- 커밋 후 `origin/main`으로 push함.

## 제외 대상

`.gitignore` 기준으로 아래 유형은 업로드하지 않음.

- `credentials.json`
- `token.json`
- `client_secret*.json`
- `service-account*.json`
- `.env`, `.env.*`
- `*.key`, `*.pem`, `*.p12`, `*.pfx`
- `*.sqlite3`, `*.db`
- `어머니 인생/`
- `한방다운로드*/`
- `개인/`, `private/`

## 자동 실행

Codex 자동화에서 평일 오전 5시 한국시간에 실행함.

자동화가 수행할 일:

- 최신 시장뉴스 확인
- 템플릿 기준으로 40개 기사 취합
- `시장뉴스/YYYY-MM-DD_시장뉴스.md` 저장
- 민감파일 제외 확인
- GitHub `main` 브랜치에 커밋 및 push

## 수동 점검 명령

```powershell
git status --short --branch
git ls-files | Select-String -Pattern 'credentials|token|\.env|sqlite|\.db|secret|key'
```
